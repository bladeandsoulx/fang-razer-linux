# v0.9.4 GitHub Release Notes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the live `v0.9.4` GitHub release's bare changelog link with the approved beginner-friendly, tag-accurate release notes.

**Architecture:** Write the approved body to one temporary Markdown file, pass that file to GitHub CLI's release editor, then read the release back through the GitHub API and compare the live body byte-for-byte. The tag, release assets, published status, and prerelease status remain unchanged.

**Tech Stack:** GitHub-flavored Markdown, GitHub CLI

## Global Constraints

- Target only `bladeandsoulx/fang-razer-linux` release `v0.9.4`.
- Describe only changes contained in the immutable `v0.9.4` tag.
- Do not mention the later README rewrite or upcoming neon installer banner.
- Keep the release published and stable; do not edit its tag or assets.
- Retain the full `v0.9.3...v0.9.4` changelog comparison link.

### Task 1: Update and verify the live release text

**Files:**
- Create temporarily: `/tmp/fang-v0.9.4-release-notes.md`
- Reference: `docs/superpowers/specs/2026-07-23-v0.9.4-release-notes-design.md`
- Modify externally: GitHub release `bladeandsoulx/fang-razer-linux@v0.9.4`

**Interfaces:**
- Consumes: The exact approved Markdown body and the existing published GitHub release.
- Produces: A published, non-prerelease `v0.9.4` release with title `v0.9.4` and the approved body.

- [ ] **Step 1: Confirm the target release before editing**

Run:

```bash
gh release view v0.9.4 \
  --repo bladeandsoulx/fang-razer-linux \
  --json tagName,name,isDraft,isPrerelease,url,body
```

Expected: tag and name are `v0.9.4`, both status flags are `false`, the URL ends
in `/releases/tag/v0.9.4`, and the current body contains only the full changelog

- [ ] **Step 2: Create the exact approved temporary release body**

Create `/tmp/fang-v0.9.4-release-notes.md` with this exact content:

````markdown
# Fang v0.9.4 — One-command Linux installer

Installing Fang is now as simple as opening Terminal and pasting one line:

```bash
curl -fsSL https://github.com/bladeandsoulx/fang-razer-linux/releases/latest/download/install.sh | bash
```

Run it as your normal desktop user—do not add `sudo`. The installer asks for
your password only when it is ready to install the verified packages.

## What’s new

- One-command installation and upgrades for supported x86_64 Linux systems.
- Support for Ubuntu 22.04/24.04, Debian 12/13, and Fedora 43/44.
- Compatible Ubuntu, Debian, and Fedora derivatives are detected automatically
  and shown a compatibility notice.
- Checksums, package metadata, app/daemon versions, and downgrade safety are
  checked before administrator access is requested.
- Fang and `fangd` are installed together, the service is enabled, and your user
  is given access when needed.
- Releases now publish as an immutable six-file set: installer, checksums, two
  DEBs, and two RPMs.
- The Support screen now clearly lists BNB Smart Chain (BEP20) and Ethereum
  (ERC20) as accepted USDT networks.

If the installer says group access was added, log out and back in once before
opening Fang.

**Full Changelog:** https://github.com/bladeandsoulx/fang-razer-linux/compare/v0.9.3...v0.9.4
````

- [ ] **Step 3: Update only the release title and body**

Run:

```bash
gh release edit v0.9.4 \
  --repo bladeandsoulx/fang-razer-linux \
  --title v0.9.4 \
  --notes-file /tmp/fang-v0.9.4-release-notes.md
```

Expected: GitHub CLI prints the `v0.9.4` release URL and exits with status 0.

- [ ] **Step 4: Verify the live release**

Run:

```bash
gh release view v0.9.4 \
  --repo bladeandsoulx/fang-razer-linux \
  --json body,tagName,name,isDraft,isPrerelease,url |
node -e 'const fs=require("fs"); const chunks=[]; process.stdin.on("data",chunk=>chunks.push(chunk)); process.stdin.on("end",()=>{const release=JSON.parse(Buffer.concat(chunks)); const expected=fs.readFileSync("/tmp/fang-v0.9.4-release-notes.md","utf8"); const valid=release.body===expected && release.tagName==="v0.9.4" && release.name==="v0.9.4" && release.isDraft===false && release.isPrerelease===false && release.url==="https://github.com/bladeandsoulx/fang-razer-linux/releases/tag/v0.9.4"; if(!valid){console.error(JSON.stringify(release)); process.exit(1);} console.log(JSON.stringify({verified:true,tagName:release.tagName,name:release.name,isDraft:release.isDraft,isPrerelease:release.isPrerelease,url:release.url,bodyBytes:Buffer.byteLength(release.body)}));});'
```

Expected: The command prints a JSON object containing `"verified":true`,
`"tagName":"v0.9.4"`, both status flags set to `false`, the exact release URL,
and `"bodyBytes":1307`, then exits with status 0.
